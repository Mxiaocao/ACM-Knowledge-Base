## 题意简述

给定一个长度为 `n` 的小写字母字符串 `s`。

每次可以把某个位置改成任意小写字母，要求在不超过：

$$
2\lceil \sqrt n \rceil + 5
$$

次修改内，把原字符串变成一个 Lyndon word。

Lyndon word 的定义是：

一个非空字符串严格小于它的所有非空真后缀。

也就是说，对于任意：

$$
1 \le i < n
$$

都要满足：

$$
s < s[i \dots n-1]
$$

题目只要求输出任意一个满足条件的字符串，因此本题是构造题。

## 第一反应

一开始的直觉是：

* 把最后一位改成 `z`；
* 尽量让前面出现一段 `a`；
* 通过“开头小、结尾大”的方式构造 Lyndon word。

这个方向是对的，但最开始还没有形成一个能覆盖所有后缀的稳定构造。

当时容易误以为：

```text
只要开头够小，最后一位是 z，就差不多可以。
```

但 Lyndon word 要比较的是：

```text
整个字符串 vs 每一个真后缀
```

所以必须保证任意后缀都能被压住，而不是只看最后一个后缀。

## 暴力思路

一个最直接的 Lyndon word 构造是：

```text
aaaa...az
```

也就是：

$$
a^{n-1}z
$$

例如：

```text
aaaaz
```

这个字符串一定是 Lyndon word。

原因是：任意真后缀都比原串少一些开头的 `a`，在比较过程中，原串最终会在某一位遇到 `a`，而后缀对应位置遇到 `z`，因此原串更小。

但是这个构造的问题是修改次数太多。

如果原串很乱，直接改成：

$$
a^{n-1}z
$$

最坏需要：

$$
O(n)
$$

次修改。

而题目只允许：

$$
O(\sqrt n)
$$

次修改。

所以不能直接把整个字符串改成标准形态。

## 核心思路

令：

$$
m = \lceil \sqrt n \rceil
$$

构造分成两部分：

第一部分，把前面一小段改成 `a`：

```text
前 min(m, n - 1) 个位置改成 a
```

第二部分，在后面周期性放 `z`：

```text
从位置 m 开始，每隔 m 个位置放一个 z
最后一位也保证是 z
```

也就是构造出这样一种结构：

```text
前面一段 a
后面每隔 m 个位置有一个 z
末尾一定是 z
```

这样做的目的：

* 前 `m` 个位置提供足够小的前缀；
* 每隔 `m` 个位置放一个 `z`，保证任意后缀在开头不远处就能遇到一个 `z`；
* 比较原串和任意后缀时，原串在对应位置是 `a`，后缀在对应位置是 `z`，所以原串严格更小。

本题的关键不是“让整个字符串都变成很规整”，而是只用少量修改制造足够多的比较胜点。

## 为什么这样一定是 Lyndon word

构造完成后的字符串记为 `t`。

需要证明：

$$
t < t[i \dots n-1]
$$

对所有：

$$
1 \le i < n
$$

都成立。

因为我们从位置 `m` 开始每隔 `m` 个位置放一个 `z`，并且强制让最后一位也是 `z`，所以对于任意一个后缀 `t[i ... n-1]`，从它的开头往后看，不超过 `m` 个位置，一定能遇到一个 `z`。

设这个 `z` 在后缀中的相对位置是 `k`，那么：

$$
0 \le k < m
$$

并且：

$$
t[i+k] = z
$$

而原串 `t` 的前 `m` 个位置被改成了 `a`，所以：

$$
t[k] = a
$$

比较原串 `t` 和后缀 `t[i ... n-1]`：

* 如果在第 `k` 位之前，原串已经比后缀小，那么已经满足要求；
* 如果前面都相等，那么到第 `k` 位时：

```text
原串：a
后缀：z
```

于是有：

$$
t < t[i \dots n-1]
$$

因此 `t` 严格小于所有非空真后缀，是 Lyndon word。

## 修改次数为什么够

修改次数分成两部分。

第一部分：把前 `min(m, n - 1)` 个位置改成 `a`。

最多修改：

$$
m
$$

次。

第二部分：从位置 `m` 开始，每隔 `m` 个位置放一个 `z`，并且最后一位也放 `z`。

这部分最多大约是：

$$
\frac{n}{m} + 1
$$

次。

因为：

$$
m = \lceil \sqrt n \rceil
$$

所以：

$$
\frac{n}{m} \le m
$$

因此总修改次数不超过：

$$
m + m + 1 = 2m + 1
$$

题目允许：

$$
2m + 5
$$

次修改，所以一定满足限制。

## 关键知识点

### 1. Lyndon word 的比较对象是所有真后缀

本题不能只保证字符串小于某一个后缀。

需要对所有：

$$
1 \le i < n
$$

都满足：

$$
s < s[i \dots n-1]
$$

所以构造时必须考虑任意后缀，而不是只看末尾字符。

### 2. 字典序比较可以通过一个关键位置决定

如果比较两个字符串时，前面若干位置都相同，那么第一个不同的位置决定大小。

本题构造的目的就是制造这样的关键位置：

```text
原串对应位置是 a
后缀对应位置是 z
```

于是原串一定更小。

### 3. 分块构造可以覆盖所有后缀

每隔 `m` 个位置放一个 `z`，意味着任意起点出发，在接下来不到 `m` 个位置内都能遇到一个 `z`。

这正好配合前 `m` 个位置的 `a`，形成比较优势。

### 4. `ceil(sqrt(n))` 用来平衡两类修改次数

本题修改次数大致是：

$$
m + \frac{n}{m}
$$

当：

$$
m \approx \sqrt n
$$

时，这个值是：

$$
O(\sqrt n)
$$

所以取：

$$
m = \lceil \sqrt n \rceil
$$

刚好满足题目的修改次数限制。

## 为什么我会卡住

这题的主要卡点有两个。

第一个卡点是：没有第一时间把 Lyndon word 的条件转化为：

```text
原串必须小于每一个真后缀
```

一开始容易只盯着末尾 `z`，但末尾是 `z` 并不能自动保证所有后缀都比原串大。

第二个卡点是：最开始想根据原串里已有的连续 `a` 来构造。

但原串已有结构并不重要。

这题真正重要的是主动制造结构：

```text
前 m 个 a + 每隔 m 个 z
```

构造题里，与其顺着原串已有特征修修补补，不如直接构造一个方便证明的目标形态。

## 正确做法步骤

1. 读入 `n` 和字符串 `s`。

2. 如果：

```cpp
n == 1
```

则直接输出原串。

长度为 `1` 的字符串没有非空真后缀，所以本身就是 Lyndon word。

3. 计算：

$$
m = \lceil \sqrt n \rceil
$$

4. 把前：

```text
min(m, n - 1)
```

个位置改成 `a`。

不能无脑把前 `m` 个都改成 `a`，因为当 `n = 2` 时，如果把最后一位也改成 `a`，可能得到：

```text
aa
```

而 `aa` 不是 Lyndon word。

5. 从位置 `m` 开始，每隔 `m` 个位置改成 `z`。

6. 强制：

```cpp
s[n - 1] = 'z';
```

7. 输出修改后的字符串。

## 代码实现要点

1. `n == 1` 必须特判。

2. 前缀改 `a` 时写：

```cpp
for(int i = 0;i < min(m, n - 1);++i)
```

而不是直接写：

```cpp
for(int i = 0;i < m;++i)
```

3. 周期性放 `z` 时写：

```cpp
for(int i = m;i < n;i += m)
```

4. 最后单独保证：

```cpp
s[n - 1] = 'z';
```

这样可以处理最后一段长度不足 `m` 的情况。

5. 本题按当前代码是单测。如果原题实际有多组测试，需要在 `main` 中读入 `t`。

## 参考代码

```cpp
#include <bits/stdc++.h>
#define code using
#define by namespace
#define Mxiaocao std
code by Mxiaocao;

using ll = long long;

int sqrt(int x){
    int m = 1;
    while(m * m < x){
        m++;
    }
    return m;
}

void solve(){
    int n; cin >> n;
    string s; cin >> s;

    if(n == 1){
        cout << s << '\n';
        return;
    }

    int m = sqrt(n);

    for(int i = 0;i < min(m, n - 1);++i){
        s[i] = 'a';
    }

    for(int i = m;i < n;i += m){
        s[i] = 'z';
    }

    s[n - 1] = 'z';

    cout << s << '\n';
    return;
}

signed main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t = 1;
    while(t--){
        solve();
    }
    return 0;
}
```

## 复杂度

设：

$$
m = \lceil \sqrt n \rceil
$$

前缀改 `a` 的次数最多是：

$$
m
$$

周期性放 `z` 的次数最多是：

$$
\frac{n}{m} + 1
$$

实际代码遍历的次数也是这些修改位置的数量级。

时间复杂度：

$$
O(\sqrt n)
$$

空间复杂度：

$$
O(1)
$$

不考虑输入字符串本身占用的空间。

## 本题总结

这题是一个很典型的构造题。

它的核心不是复杂算法，而是构造一个能同时满足两个条件的结构：

```text
1. 能保证 Lyndon word 条件；
2. 修改次数不超过 O(sqrt(n))。
```

构造方式是：

```text
前 m 个位置放 a
后面每隔 m 个位置放 z
末尾保证 z
```

其中：

$$
m = \lceil \sqrt n \rceil
$$

前 `m` 个 `a` 负责让原串开头足够小；周期性的 `z` 负责让任意真后缀在很短距离内遇到大字符；`sqrt(n)` 负责平衡前缀修改次数和周期标记次数。

这题二刷时不能只背代码，重点要能重新推出这个构造，并能说明为什么任意后缀都会被压住。
