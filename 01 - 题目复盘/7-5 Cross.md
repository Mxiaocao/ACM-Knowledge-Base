## 题意简述

有 `n` 轮攻击，初始参数：

```text
r = 0, c = 0
```

第 `i` 轮依次发生：

1. `r += a[i]`
2. 可以选择施放一次技能，全程最多使用 `K` 次：
   - `c += 1`
   - 如果 `r > 0`，则 `r -= 1`
   - 如果 `r = 0`，技能仍然可以使用，只是不能减少 `r`
3. 目标回复 `r * R` 点血量
4. 我方造成 `X` 点固定伤害
5. 目标受到 `c * C` 点持续伤害

要求最大化目标总血量下降值。这个值可以为负，表示目标最终反而回血更多。

每一回合的净贡献为：

$$
X + cC - rR
$$

所以问题可以理解为：

> 在最多使用 `K` 次技能的前提下，合理安排技能使用回合，使所有回合的总贡献最大。

数据范围：

```text
1 <= K <= n <= 2 * 10^6
0 <= X, R, C, a[i] <= 10^6
```

## 第一反应

一开始看到这题的时候，感觉它像一个带状态的多轮决策问题。

因为每一轮都有两个会延续到后面的变量：

- `r` 会影响后续每一轮的回血量；
- `c` 会影响后续每一轮的持续伤害。

## 核心思路

先把问题分成两部分：

1. 完全不用技能时的基础贡献 `base`
2. 每次使用技能带来的额外收益 `gain`

最终答案为：

$$
answer = base + \text{最大的 K 个技能收益之和}
$$

### 1. 不用技能时的基础贡献

如果完全不用技能，则 `c = 0`，每轮只有固定伤害和敌人回血。

第 `i` 轮：

```text
r += a[i]
贡献 = X - r * R
```

所以：

$$
base = \sum_{i=1}^{n} (X - r_iR)
$$

其中 `r_i` 是第 `i` 轮加上 `a[i]` 后的恢复层数。

### 2. 一次技能的收益

如果在第 `i` 轮使用技能，它至少会让：

```text
c += 1
```

从第 `i` 轮到第 `n` 轮，每一轮都会多造成 `C` 点持续伤害。

所以这部分收益一定是：

$$
(n - i + 1)C
$$

如果使用技能时 `r > 0`，它还能让：

```text
r -= 1
```

这相当于从第 `i` 轮到第 `n` 轮，每轮敌人少回复 `R` 点血量。

所以额外收益为：

$$
(n - i + 1)R
$$

因此，一次技能有两种情况：

- 如果当前 `r > 0`：

$$
gain_i = (n - i + 1)(C + R)
$$

- 如果当前 `r = 0`：

$$
gain_i = (n - i + 1)C
$$

### 3. 为什么可以模拟每一轮都尝试使用技能

关键观察：

> 如果某一轮当前 `r > 0`，那么此时使用技能一定比拖到后面使用更优。

因为现在使用可以让 `c += 1` 和 `r -= 1` 的影响持续更多回合，收益一定不比后面差。

所以我们可以假装：

```text
每一轮都尝试使用一次技能
```

然后模拟出每一轮“如果这轮占用一个技能名额，最多能获得多少边际收益”。

具体模拟：

```text
r = 0

for i = 1 到 n:
    r += a[i]

    if r > 0:
        gain[i] = (n - i + 1) * (C + R)
        r -= 1
    else:
        gain[i] = (n - i + 1) * C
```

这样得到的 `gain[i]` 就表示：

> 第 `i` 轮使用技能所能带来的边际贡献。

由于最多使用 `K` 次技能，所以只需要从所有 `gain[i]` 中选最大的 `K` 个。

更准确地说，技能的收益可以拆成两部分：

1. `c += 1` 带来的持续伤害收益；
2. 如果 `r > 0`，`r -= 1` 带来的减少回血收益。

对于第 1 部分，显然越早使用，影响的回合数越多。

对于第 2 部分，可以把每一点 `r` 看成一个“等待被消掉的回血层数”。如果某一轮已经有可消掉的 `r`，那么现在消掉它，一定不比未来再消掉差。因为现在消掉会让后面更多回合少回血。

所以在构造所有可能的“成功削掉 r 的机会”时，可以用一种最激进的方式模拟：

> 每一轮都尝试使用技能，能削 `r` 就立刻削。

这样得到的是每个技能名额可能产生的最大边际收益序列。
最后我们不是真的每轮都用技能，而是从这些边际收益里选最大的 `K` 个。

## 关键知识点

### 1. 技能不会产生负收益

技能效果：

```text
c += 1
r -= 1，如果 r > 0
```

`c += 1` 一定不亏，`r -= 1` 也只会减少敌人回血。

所以技能没有负面效果，最优方案可以认为一定使用满 `K` 次。

### 2. 只看 `c += 1`，技能越早用越好

第 `i` 轮使用技能，持续伤害收益为：

$$
(n - i + 1)C
$$

显然 `i` 越小，收益越大。

### 3. 如果当前 `r > 0`，技能也越早用越好

如果当前 `r > 0`，使用技能可以同时获得：

$$
(n - i + 1)C
$$

和：

$$
(n - i + 1)R
$$

总收益为：

$$
(n - i + 1)(C + R)
$$

拖到后面只会让影响回合数变少，所以不如现在用。

### 4. `r = 0` 时也可能提前空放技能

当 `r = 0` 时，技能只能增加持续伤害，不能减少回血层数。

但是如果未来很久之后才出现 `r > 0`，那么提前空放技能可能仍然更赚。

例如：

```text
当前空放收益 = (n - i + 1) * C
未来削 r 收益 = (n - j + 1) * (C + R)
```

所以不能简单写成：

```cpp
if(r > 0) 用技能;
else 不用技能;
```

正确做法是把所有回合的技能边际收益都算出来，然后选最大的 `K` 个。

## 为什么我会卡住

没有把技能收益拆开：

1. `r` 和 `c` 都会影响后续回合，状态看起来有后效性。
2. 没意识到一次技能的影响可以拆成：
   - 持续伤害收益
   - 减少回血收益
3. 没意识到可以先算不用技能的 `base`，再单独算技能的边际收益。
4. 没想到最后其实是一个 TopK 问题。

本题真正的突破口是：

> 不要直接思考“怎么安排技能”，而是先思考“每一次技能最多能带来多少额外收益”。

## 正确做法步骤

### 第一步：读入数据

读入：

```text
n, K, R, X, C
a[1...n]
```

注意题目本身是单组输入，没有 `t`。

### 第二步：计算不用技能时的基础贡献

```text
r = 0
base = 0

for i = 1 到 n:
    r += a[i]
    base += X - r * R
```

这里 `base` 可能超过 `long long`，所以使用 `__int128`。

### 第三步：计算每一轮使用技能的边际收益

重新模拟一遍，假设每一轮都尝试使用技能：

```text
r = 0

for i = 1 到 n:
    r += a[i]
    len = n - i + 1

    if r > 0:
        gain[i] = len * (R + C)
        r -= 1
    else:
        gain[i] = len * C
```

其中 `gain[i]` 表示第 `i` 轮使用技能带来的额外收益。

### 第四步：选最大的 K 个收益

因为最多使用 `K` 次技能，所以取最大的 `K` 个 `gain[i]` 加到答案里。

可以排序：

```cpp
sort(gain.begin(), gain.end(), greater<ll>());
```

复杂度是：

$$
O(n \log n)
$$

但更推荐使用 `nth_element`：

```cpp
nth_element(gain.begin(), gain.begin() + K, gain.end(), greater<ll>());
```

这样平均复杂度是：

$$
O(n)
$$

注意当 `K == n` 时，`gain.begin() + K == gain.end()`，不能作为 `nth_element` 的第二个参数，所以要特判：

```cpp
if(k < n){
    nth_element(gain.begin(), gain.begin() + k, gain.end(), greater<ll>());
}
```

### 第五步：输出答案

由于答案可能达到 `1e24` 量级，不能直接用 `long long`。

需要使用：

```cpp
using i128 = __int128;
```

并手写输出函数。

## 代码实现要点

1. `r` 表示恢复层数，最大大约为：

$$
2 \times 10^6 \times 10^6 = 2 \times 10^{12}
$$

所以 `r` 用 `long long` 足够。

2. 单个技能收益最大约为：

$$
2 \times 10^6 \times 2 \times 10^6 = 4 \times 10^{12}
$$

所以 `gain[i]` 用 `long long` 足够。

3. 总答案可能达到：

$$
10^{24}
$$

所以 `res` 必须使用 `__int128`。

4. 计算基础贡献时必须写成：

```cpp
res += (i128)X - (i128)R * r;
```

不能写成：

```cpp
res += X - R * r;
```

否则乘法可能先用 `long long` 算，存在溢出风险。

5. 本题没有多测，不要读入 `t`。
6. 取前 `K` 大时，`nth_element` 的写法是：

```cpp
if(k < n){
    nth_element(tmp.begin(), tmp.begin() + k, tmp.end(), greater<ll>());
}
```

前 `k` 个元素就是最大的 `k` 个，但内部不保证有序。

## 参考代码

```cpp
#include <bits/stdc++.h>
#define code using
#define by namespace
#define Mxiaocao std
code by Mxiaocao;

using ll = long long;
using i128 = __int128;

void print(i128 x){
    if(x == 0){
        cout << 0 << '\n';
        return;
    }

    if(x < 0){
        cout << '-';
        x = -x;
    }

    string s;
    while(x){
        s.push_back(char('0' + x % 10));
        x /= 10;
    }

    reverse(s.begin(), s.end());
    cout << s << '\n';
}

void solve(){
    int n, k;
    ll R, X, C;
    cin >> n >> k >> R >> X >> C;

    vector<ll> a(n);
    for(int i = 0; i < n; ++i){
        cin >> a[i];
    }

    i128 res = 0;
    ll r = 0;

    for(int i = 0; i < n; ++i){
        r += a[i];
        res += (i128)X - (i128)R * r;
    }

    vector<ll> tmp(n);
    r = 0;

    for(int i = 0; i < n; ++i){
        r += a[i];

        ll len = n - i;

        if(r > 0){
            tmp[i] = 1LL * (R + C) * len;
            r--;
        }else{
            tmp[i] = 1LL * C * len;
        }
    }

    if(k < n){
        nth_element(tmp.begin(), tmp.begin() + k, tmp.end(), greater<ll>());
    }

    for(int i = 0; i < k; ++i){
        res += (i128)tmp[i];
    }

    print(res);
}

signed main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t = 1;
    // cin >> t;

    while(t--){
        solve();
    }

    return 0;
}
```

## 复杂度

数据范围：

```text
n <= 2 * 10^6
K <= n
a[i], R, X, C <= 10^6
```

允许复杂度大致为：

$$
O(n)
$$

或：

$$
O(n \log n)
$$

本题使用 `nth_element`，平均复杂度为：

$$
O(n)
$$

空间复杂度为：

$$
O(n)
$$

其中主要空间来自：

```cpp
vector<ll> a(n);
vector<ll> tmp(n);
```

也可以进一步优化掉 `a` 的一部分存储，但由于需要两遍扫描，保留 `a` 更直观。

## 本题总结

这题表面上是一个带状态的多轮决策问题，但本质是：

> 基础贡献 + 操作边际贡献 + TopK 选择。

不要直接陷入“第几轮用技能”的枚举或 DP。

真正要抓住的是：

1. 不用技能时的贡献可以直接算。
2. 一次技能的影响可以拆成持续伤害收益和减少回血收益。
3. 如果当前能削 `r`，越早削越好。
4. 可以假装每轮都使用技能，得到每轮技能的边际收益。
5. 最后选最大的 `K` 个收益。

本题的思维价值在于：

> 把动态决策问题转化成静态收益选择问题。

这类题以后看到“操作会影响未来很多轮”时，可以优先尝试拆成边际贡献。

$$

如果 `r = 0`：

$$

5. 假装每轮都用技能，模拟出所有 `gain`。
6. 选最大的 `K` 个 `gain`。
