## 题意简述

一条街道从位置 `0` 向右延伸。

街上有：

```text id="gm50th"
n 台游戏机
n 枚金币
```

第 `i` 台游戏机的位置是 `a[i]`，第 `i` 枚金币的位置是 `b[i]`。

你从位置 `0` 出发，可以在街道上左右移动。

规则是：

* 经过金币的位置时，可以捡起金币；
* 身上可以同时携带多枚金币；
* 玩一台游戏机需要消耗一枚金币；
* 每台游戏机必须恰好玩一次；
* 问最少需要移动多少距离。

注意：如果走到某台机器时手里没有金币，并不代表失败。可以之后去右边拿金币，再回来补这台机器。

## 第一反应

一开始题意不太好理解，尤其是：

```text id="qz2vt3"
经过机器时如果没有金币怎么办？
```

后来理解到：

```text id="e4vkqq"
如果当前没金币，这台机器可以之后回来再玩。
```

所以这题不是简单从左走到右，也不是给每台机器匹配最近金币。

它本质上是一个数轴上的“欠金币 / 补金币”问题。

如果某段左边机器数量比金币数量多，那么说明左边缺金币，必须从右边拿金币回来补。

## 暴力思路

一种直接模拟思路是：

```text id="fsxfol"
遇到金币就捡
遇到机器如果有金币就玩
如果没金币，就继续往右找金币，再回来
```

但这样不好判断最优路线。

另一种暴力是枚举每台机器使用哪一枚金币，再计算移动距离。

但是 `n` 很大，机器和金币的匹配关系太多，不能枚举。

所以需要换一个角度：

```text id="b36jme"
不直接模拟人的路线，而是判断每一段相邻位置之间的路至少要被走几次。
```

## 核心思路

把所有机器和金币看成数轴上的事件。

定义：

```text id="0qr7q5"
机器：-1
金币：+1
```

把所有事件合并排序，并加入起点：

```cpp id="s3my4i"
c.push_back({0, 0});
```

然后维护前缀余额：

```text id="7mf0mc"
余额 = 已经过的金币数 - 已经过的机器数
```

如果某个位置处理完后：

```text id="bf6d6d"
余额 < 0
```

说明左边机器数量比金币数量多。

也就是：

```text id="yf2br8"
左边缺金币，必须从右边拿金币回来补。
```

于是问题就转化成：

```text id="pemykl"
对每一段相邻事件点之间的路，根据左边是否缺金币，以及最终停点在哪边，判断这段路需要走几次。
```

## 关键知识点

### 1. 机器和金币可以转成事件

机器需要消耗金币，可以记为 `-1`。

金币提供资源，可以记为 `+1`。

从左到右扫描时，前缀余额：

```text id="7ssrfk"
金币数 - 机器数
```

表示左边区域目前是“金币有余”还是“机器缺金币”。

当余额小于 `0` 时，说明左边机器比金币多，左边缺金币，需要从右边拿金币回来。

### 2. 一段路要看左端点处理后的余额

对于相邻两个事件点：

```text id="93vhx2"
c[i-1].first -> c[i].first
```

这一段路在走之前，已经处理完 `c[i-1]` 位置，但还没有到达 `c[i]`。

所以判断这段路左边是否缺金币，应该看：

```cpp id="k01fqn"
c[i - 1].second
```

而不是：

```cpp id="je9h6f"
c[i].second
```

这个点非常容易错。

例如：

```text id="ehl6r6"
机器在 1
金币在 3
```

排序后：

```text id="zy7k4h"
(0, 0)
(1, -1)
(3, 0)
```

对于路段：

```text id="u6d1fw"
0 -> 1
```

走这段路之前还没有到达位置 `1`，所以不能提前把位置 `1` 的机器算进去。

因此这一段应该看左端点 `0` 的余额，而不是右端点 `1` 的余额。

### 3. 最终停在路段右边时的代价

考虑某一段路，长度为：

```text id="g498wh"
len = c[i].first - c[i-1].first
```

如果最终停在这段路右边，那么这段路最终需要从左到右穿过去。

如果左边余额不欠金币：

```text id="33bzt5"
balance >= 0
```

那么这段路只需要走一次：

```text id="xyroox"
len
```

如果左边余额欠金币：

```text id="f1yx19"
balance < 0
```

说明必须从右边拿金币回来补左边机器。

同时最终还要停在右边，所以这段路至少要走三次：

```text id="y74rbw"
左 -> 右
右 -> 左
左 -> 右
```

代价是：

```text id="af06g9"
3 * len
```

### 4. 最终停在路段左边时的代价

如果最终停在这段路左边，那么右边的事情也要处理完，最后还要回来。

这段路至少需要：

```text id="286k1m"
左 -> 右
右 -> 左
```

所以代价统一是：

```text id="7g60v5"
2 * len
```

无论左边是否欠金币，都可以在这一来一回过程中完成金币运输。

### 5. 枚举最终停点

可以先假设最终停在 `0`。

那么所有路段都在终点右边，所有路段都需要走过去再回来，所以初始代价是：

```cpp id="fslpfk"
tmp = sum(r[i])
```

其中 `r[i]` 表示最终停在该路段左边时，这段路的代价，也就是：

```text id="55172z"
2 * len
```

然后把最终停点从左往右移动。

当终点跨过第 `i` 段路后，这段路从“终点右边”变成“终点左边”。

所以它的贡献从：

```text id="p1eswq"
r[i]
```

变成：

```text id="vvy2h2"
l[i]
```

更新：

```cpp id="u8jumz"
tmp = tmp - r[i] + l[i];
res = min(res, tmp);
```

最终所有停点取最小值。

## 为什么我会卡住

这题主要卡在三个地方。

第一个卡点是题意建模。

一开始没有完全理解：

```text id="w7gb64"
走到机器时没金币，可以之后拿金币回来补。
```

理解这个规则后，题目才变成“左边是否缺金币”的问题。

第二个卡点是余额含义。

最终采用：

```text id="lix1t8"
机器：-1
金币：+1
余额 = 金币数 - 机器数
```

所以：

```text id="v8s16v"
余额 < 0
```

表示左边缺金币。

第三个卡点是路段判断应该看哪一侧状态。

对于：

```text id="qdn933"
c[i-1] -> c[i]
```

应该看：

```cpp id="qphwxs"
c[i-1].second
```

因为走这段路之前，还没有处理 `c[i]` 位置的事件。

## 正确做法步骤

1. 读入 `n`、机器位置数组 `a`、金币位置数组 `b`。

2. 把所有事件放入数组 `c`：

```cpp id="61mg8b"
机器位置：-1
金币位置：+1
起点 0：0
```

3. 对事件按位置排序。

4. 对事件值做前缀和，使 `c[i].second` 表示处理完当前位置后的余额。

5. 枚举每一段相邻事件点：

```text id="e5hbcv"
c[i-1].first -> c[i].first
```

计算长度：

```cpp id="8f7tgx"
len = c[i].first - c[i - 1].first;
```

6. 如果左端点余额：

```cpp id="9xge4o"
c[i - 1].second >= 0
```

那么最终停在右边时，这段路走一次：

```cpp id="n97gqb"
l[i] = len;
```

否则左边缺金币，需要右边拿金币回来再过去：

```cpp id="duz7nh"
l[i] = 3 * len;
```

7. 最终停在左边时，这段路统一走两次：

```cpp id="mfa03z"
r[i] = 2 * len;
```

8. 初始假设最终停在 `0`：

```cpp id="zfdpuv"
tmp = sum(r[i]);
```

9. 从左到右枚举最终停点，每跨过一段，把该段贡献从 `r[i]` 换成 `l[i]`：

```cpp id="zo2cdr"
tmp = tmp - r[i] + l[i];
res = min(res, tmp);
```

10. 输出最小值。

## 代码实现要点

### 1. 位置和答案都要用 long long

位置最大可到：

$$
10^9
$$

事件点数量可到：

$$
2n+1
$$

总距离可能到：

$$
10^{15}
$$

级别，所以：

```cpp id="zh1w3z"
vector<ll> a, b;
vector<pair<ll,int>> c;
vector<ll> l, r;
ll tmp, res;
```

都应该使用 `long long`。

### 2. 距离必须是两个位置相减

错误写法：

```cpp id="dko0wi"
r[i] = 2 * (c[i].first - c[i - 1].second);
```

这里把位置和余额混在一起了。

正确写法：

```cpp id="y5omd1"
r[i] = 2 * (c[i].first - c[i - 1].first);
```

### 3. 一段路看左端点余额

正确写法：

```cpp id="v04349"
if(c[i - 1].second >= 0)
```

不要写成：

```cpp id="d559fe"
if(c[i].second >= 0)
```

因为这会提前处理右端点事件。

### 4. 同位置事件不需要特殊处理

如果机器和金币在同一个位置，排序后可能出现长度为 `0` 的路段。

这些路段贡献为 `0`，不会影响答案。

最后离开这个位置前，前缀余额已经包含该位置的所有事件，所以整体逻辑仍然成立。

## 参考代码

```cpp id="fnz6ox"
#include <bits/stdc++.h>
#define code using
#define by namespace
#define Mxiaocao std
code by Mxiaocao;

using ll = long long;

void solve(){
    int n; cin >> n;
    vector<ll> a(n), b(n);

    for(int i = 0;i < n;++i) cin >> a[i];
    for(int i = 0;i < n;++i) cin >> b[i];

    vector<pair<ll,int>> c;
    c.reserve(2 * n + 10);

    for(int i = 0;i < n;++i){
        c.push_back({a[i], -1});
        c.push_back({b[i], 1});
    }

    c.push_back({0, 0});
    sort(c.begin(), c.end());

    for(int i = 1;i < 2 * n + 1;++i){
        c[i].second += c[i - 1].second;
    }

    vector<ll> l(2 * n + 10), r(2 * n + 10);

    for(int i = 1;i < 2 * n + 1;++i){
        ll len = c[i].first - c[i - 1].first;

        if(c[i - 1].second >= 0){
            l[i] = len;
        }else{
            l[i] = 3 * len;
        }

        r[i] = 2 * len;
    }

    ll tmp = 0;

    for(int i = 1;i < 2 * n + 1;++i){
        tmp += r[i];
    }

    ll res = tmp;

    for(int i = 1;i < 2 * n + 1;++i){
        tmp = tmp - r[i] + l[i];
        res = min(res, tmp);
    }

    cout << res << '\n';
    return;
}

signed main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t = 1;
    // cin >> t;
    while(t--){
        solve();
    }
    return 0;
}
```

## 复杂度

一共有：

$$
2n + 1
$$

个事件点。

排序复杂度：

$$
O(n \log n)
$$

扫描和枚举最终停点复杂度：

$$
O(n)
$$

总时间复杂度：

$$
O(n \log n)
$$

空间复杂度：

$$
O(n)
$$

由于：

$$
n \le 10^6
$$

可以通过。

如果题目保证两个数组本身有序，也可以用双指针合并事件做到：

$$
O(n)
$$

但当前写法更直接，且更适合复盘理解。

## 本题总结

这题是一个数轴建模题。

它的关键不是模拟路线，而是把问题改写成：

```text id="xamf25"
每一段相邻事件点之间的路，最少要被走几次？
```

通过机器记 `-1`、金币记 `+1`，前缀余额就能表示左边是否缺金币。

当某段路左边余额为负时，说明左边缺金币，必须从右边把金币带回来补。

如果最终停在这段路右边，那么欠金币路段要走三次；如果最终停在这段路左边，那么这段路统一走两次。

最后枚举最终停点，用贡献切换：

```cpp id="g50qci"
tmp = tmp - r[i] + l[i];
```

得到最小答案。

这题二刷的重点是重新推导 `l[i] / r[i]` 的含义，而不是背数组名。
