## 题意简述

给定一个长度为 $n$ 的非递减数组 $a$。

每个元素满足：

$$
a_i=-1
$$

或者：

$$
a_i>0
$$

需要统计有多少个下标序列：

$$
1\le i_1<i_2<\cdots<i_k\le n
$$

使得对应子序列的交替和为零：

$$
a_{i_1}-a_{i_2}+a_{i_3}-a_{i_4}+\cdots=0
$$

空子序列也被认为交替和为零。

答案对：

$$
10^9+7
$$

取模。

## 错误思路

### 看到计数就直接猜 DP

计数问题不等于一定需要 DP。

在没有刻画合法子序列结构前设计 DP，导致状态始终不清楚。

### 使用 dp[i][j] 记录起点和终点

固定首尾下标后，中间仍然可以选择不同元素，因此无法确定交替和。

状态数量也至少为：

$$
O(n^2)
$$

### 保存所有可能交替和

交替和值域很大，无法作为可接受的状态维度。

### 只改变遍历方向

从前往后或者从后往前，都没有消除交替和状态过多的问题。

### 错误理解奇数个 -1 的结构

我一度认为奇数个 `-1` 时只能出现两种不同正整数。

反例：

```text
1 1 2 3 5 5
```

正确限制是：

```text
两两分组后恰好一对相差 1，
其余每一对相等。
```

## 核心思路

由于数组非递减，所有 `-1` 都位于正整数之前。

设选择了 $r$ 个 `-1`，按照 $r$ 的奇偶性分类。

### 选择偶数个 -1

偶数个 `-1` 的交替和为零，后面的正整数从加号开始。

设正整数子序列为：

$$
b_1\le b_2\le\cdots\le b_t
$$

若 $t$ 为奇数：

$$
b_1+(b_3-b_2)+(b_5-b_4)+\cdots>0
$$

不可能等于零。

若 $t$ 为偶数：

$$
(b_1-b_2)+(b_3-b_4)+\cdots
$$

每一项都小于等于零。

总和为零时，每一项必须为零，所以：

$$
b_1=b_2,\quad b_3=b_4,\quad\cdots
$$

等价于每一种正整数都选择偶数个。

### 选择奇数个 -1

奇数个 `-1` 的交替和为 $-1$，后面的正整数从减号开始。

正整数部分必须贡献 $1$。

正整数数量必须为偶数，此时贡献为：

$$
(b_2-b_1)+(b_4-b_3)+\cdots
$$

每一项都是非负整数。

若总和为 $1$，则恰好一项为 $1$，其余项全部为零。

所以正整数对具有：

```text
(x,x) (y,y+1) (z,z) ...
```

的结构。

等价于：

* 某两个相差 $1$ 的正整数值分别选择奇数个；
* 其他正整数值全部选择偶数个。

## 偶数选择与奇数选择计数

某个值出现 $c$ 次。

从中选择偶数个和选择奇数个的方案数都为：

$$
2^{c-1}
$$

因为翻转某个固定位置的选择状态，可以在奇数方案和偶数方案之间建立一一对应。

## 统计量

定义：

* $p$：`-1` 的数量；
* $s$：正整数总数；
* $m$：不同正整数种类数；
* $q$：不同正整数中相差恰好为 $1$ 的相邻数值对数量。

固定每一种正整数所需选择次数的奇偶性后，正整数部分方案数为：

$$
2^{s-m}
$$

## 最终公式

如果：

$$
p=0
$$

则：

$$
\text{Ans}=2^{s-m}
$$

如果：

$$
p>0
$$

从 `-1` 中选择偶数个和选择奇数个的方案数都是：

$$
2^{p-1}
$$

所以：

$$
\text{Ans}=
\begin{cases}
2^{s-m}, & p=0,\\[6pt]
(q+1)\times2^{p+s-m-1}, & p>0.
\end{cases}
$$

## 一次扫描统计

由于数组非递减：

* 相同正整数连续出现；
* 不同正整数递增出现。

扫描时：

* 遇到 `-1`，令 $p$ 加一；
* 遇到正整数，令 $s$ 加一；
* 当前值与 `last` 不同时，令 $m$ 加一；
* 如果当前新值等于 `last+1`，令 $q$ 加一。

## 为什么我会卡住

我看到计数后直接开始设计 DP，没有先刻画合法子序列结构。

非递减条件使两两分组后的每一项符号固定，合法结构实际上非常有限。

## 正确做法步骤

1. 统计 $p,s,m,q$；
2. 按照选择 `-1` 的数量奇偶性刻画正整数结构；
3. 将结构转成每个数值组的选择奇偶性；
4. 使用偶数子集和奇数子集都为 $2^{c-1}$；
5. 代入最终公式。

## 代码实现要点

* 预处理二的幂；
* `last` 记录上一个不同正整数；
* 只有出现新值时才更新 $m$ 和 $q$；
* 空子序列已经包含在公式中；
* 使用 `long long` 进行乘法并取模。

## 参考代码

以下代码是在理解题解后独立整理并提交通过的实现。

```cpp
#include <bits/stdc++.h>
#define code using
#define by namespace
#define Mxiaocao std
code by Mxiaocao;

using ll = long long;
const int mod = 1e9+7;
const int N = 2e5+5;

ll pow2[N];

void solve(){
    int n; cin >> n;
    int p = 0,s = 0,m = 0,q = 0;
    ll last = -1;
    bool ok = 0;

    for(int i = 0;i < n;++i){
        ll x; cin >> x;

        if(x == -1){
            p++;
            continue;
        }

        s++;

        if(!ok || x != last){
            if(ok && x == last+1){
                q++;
            }
            m++;
            last = x;
            ok = true;
        }
    }

    ll res = 0;
    if(p == 0){
        res = pow2[s-m];
    }else{
        res = pow2[p+s-m-1]*(q+1)%mod;
    }

    cout << res << '\n';
    return;
}

signed main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t = 1;
    cin >> t;

    pow2[0] = 1;
    for(int i = 1;i < N;++i){
        pow2[i] = pow2[i-1]*2%mod;
    }

    while(t--){
        solve();
    }
    return 0;
}
```
