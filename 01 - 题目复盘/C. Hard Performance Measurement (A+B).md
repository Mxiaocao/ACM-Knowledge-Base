## 题意简述

这是一道交互式构造题。

交互器先给出一个整数 $n$。

需要先输出：

- 一个正整数 $x$；
- 一个严格递增的正整数数组 $a_1,a_2,\dots,a_n$。

要求初始数组中不存在两个不同元素满足：

$$
a_i+a_j=x
$$

之后交互器给出一个位置 $p$，需要只修改 $a_p$，并保证修改后：

1. 数组仍然严格递增；
2. 数组中存在两个不同元素之和等于 $x$。

## 第一反应

我现场第一反应是这题像一道签到构造题，而且样例看起来很有规律。

样例中出现了类似：

$$
1,4,7,10
$$

这样的数组，相邻元素差为 $3$，所以我一开始自然受到样例影响，往等差构造方向想。

但是我一开始没有真正想清楚为什么要差 $3$，也没有完全从题目的核心矛盾出发分析：

1. 初始数组里不能存在两数和为 $x$；
2. 修改任意一个指定位置后，又必须能制造出两数和为 $x$；
3. 修改后数组还必须严格递增。

也就是说，我一开始更像是在模仿样例表面规律，而不是从构造目标反推不变量。

## 错误思路

第一发 WA 主要是被样例带偏。

当时我关注到样例里某些输出像是把靠后的某个数组元素当成目标值 $x$，例如关注到了类似 $a_{n-1}$ 这样的值，然后想让前面的数围绕这个值去凑。

我的想法更像是：

- 先确定一个比较靠后的值作为 $x$；
- 前面若干个数尝试安排成能在修改后凑出 $x$；
- 但是后面一两个位置会出现不好处理的情况。

这个方向的问题是，它只适配样例中的某些位置，不保证交互器给任意 $p$ 时都能修改成功。

交互构造题不能只让样例中的那个位置成立，而是要让所有可能被交互器选择的位置都有合法修改方案。

## 核心思路

这题的关键不是记住样例里的数组，而是从构造目标反推。

如果希望交互器给出位置 $p$ 后，修改 $a_p$ 能和另一个数凑成 $x$，可以先设：

$$
new\_a_p+a_q=x
$$

如果统一让：

$$
new\_a_p=a_p+1
$$

那么就需要构造数组，使得对大多数位置 $p$，都存在一个位置 $q$ 满足：

$$
a_p+1+a_q=x
$$

这提示可以设计一种对称配对。

令：

$$
a_i=3i-2
$$

也就是：

$$
1,4,7,10,\dots
$$

再令：

$$
x=3n
$$

于是：

$$
a_p+1+a_{n-p+1}
=
(3p-2)+1+(3(n-p+1)-2)
$$

化简得到：

$$
a_p+1+a_{n-p+1}=3n=x
$$

所以，除了对称位置等于自己的情况外，只要把 $a_p$ 改成 $a_p+1$，它就能和 $a_{n-p+1}$ 配对。

## 为什么初始数组没有两数和为 x

构造为：

$$
a_i=3i-2
$$

所以所有初始元素都满足：

$$
a_i\equiv 1 \pmod 3
$$

任意两个不同初始元素相加，都会满足：

$$
a_i+a_j\equiv 2 \pmod 3
$$

而：

$$
x=3n\equiv 0 \pmod 3
$$

因此初始数组中不可能存在两个不同元素之和等于 $x$。

这一步是整道题的"防守条件"：先保证初始状态一定安全。

## 为什么修改后仍然严格递增

数组相邻两项差为 $3$：

$$
a_{i+1}-a_i=3
$$

普通情况下把：

$$
a_p
$$

修改成：

$$
a_p+1
$$

由于左右之间本来留了空位，所以仍然有：

$$
a_{p-1}<a_p+1<a_{p+1}
$$

因此修改不会破坏严格递增。

这也是差 $3$ 的真正作用之一：给单点修改留出安全空间。

## 奇数中点特判

普通情况下，修改第 $p$ 个数后，它和对称位置：

$$
n-p+1
$$

配对。

但是如果：

$$
p=n-p+1
$$

则有：

$$
p=\frac{n+1}{2}
$$

这只会在 $n$ 为奇数，并且 $p$ 是中点时出现。

此时如果仍然把 $a_p$ 改成 $a_p+1$，它需要配对的位置还是自己。但题目要求两个不同元素，不能自己和自己配对。

所以需要特判奇数中点。

当 $n$ 为奇数且 $p=\frac{n+1}{2}$ 时，可以把 $a_p$ 改成：

$$
x-a_{p+1}
$$

这样新的 $a_p$ 可以和右边的 $a_{p+1}$ 配对：

$$
new\_a_p+a_{p+1}=x
$$

同时这个新值仍然夹在 $a_{p-1}$ 和 $a_{p+1}$ 之间，不会破坏严格递增。

## 正确做法步骤

1. 构造数组：$a_i=3i-2$，也就是：$1,4,7,10,\dots$
2. 令：$x=a_n+2=3n$
3. 初始时所有 $a_i\equiv 1 \pmod 3$，任意两数和都 $\equiv 2 \pmod 3$，而 $x\equiv 0 \pmod 3$，所以初始没有两数和为 $x$。
4. 读入交互器给出的位置 $p$。
5. 如果不是奇数中点，输出：$a_p+1$，此时它可以和 $a_{n-p+1}$ 配对。
6. 如果 $n$ 为奇数且 $p$ 是中点，输出：$x-a_{p+1}$，让它和右边的 $a_{p+1}$ 配对。

## 代码实现要点

- 数组使用 $1$ 下标更方便处理对称位置。
- 构造时从 $1$ 开始，每次加 $3$。
- $x=a_n+2$。
- 普通位置输出 $a[p]+1$。
- 奇数中点输出 $x-a[p+1]$。
- 交互题输出后需要刷新。这里使用 `endl` 可以自动刷新。
- 非多测题，不需要读 $t$。

## 参考代码

```cpp
#include <bits/stdc++.h>
#define code using
#define by namespace
#define Mxiaocao std
code by Mxiaocao;

using ll = long long;

void solve(){
    int n; cin >> n;
    vector<int> a(n+1);

    a[1] = 1;
    for(int i = 2;i <= n;++i){
        a[i] = a[i-1] + 3;
    }

    int x = a[n] + 2;

    cout << x << endl;
    for(int i = 1;i <= n;++i){
        if(i > 1) cout << ' ';
        cout << a[i];
    }
    cout << endl;

    int p; cin >> p;

    if(n % 2 == 1 && p == (n+1)/2){
        cout << x - a[p+1] << endl;
    }else{
        cout << a[p] + 1 << endl;
    }
}

signed main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    solve();
    return 0;
}
```

构造数组需要：$O(n)$

回答交互位置需要：$O(1)$

总复杂度：$O(n)$

## 本题总结

这题的核心不是"样例用了差 $3$"，而是差 $3$ 同时服务于两个目标：

1. 初始所有元素同余于 $1 \pmod 3$，避免任意两数和等于 $x$；
2. 相邻元素之间留出空位，使得 $a_p$ 可以安全改成 $a_p+1$。

真正的构造逻辑是：

```
先让初始状态一定没有两数和为 x，
再让每个位置修改后都能和对称位置配对成 x。
```

这题给我的提醒是：

构造题不能只从样例反推规律，而要从题目要求反推不变量。

如果一个构造需要交互器任选位置，那么必须验证所有位置，而不是只验证样例出现的位置。

## 二刷提醒
