## 题意简述

给定一个长度为 `n` 的数组 `a`，数组元素只可能是 `1、2、3`。

要求把数组切成三个连续的非空部分：

```text
[左段] [中段] [右段]
```

左段要求：

```text
cnt1 >= cnt2 + cnt3
```

也就是左段中 `1` 的数量至少要压过 `2` 和 `3` 的总数量。

中段要求：

```text
cnt1 + cnt2 >= cnt3
```

也就是中段中 `1` 和 `2` 的数量总和至少要压过 `3` 的数量。

右段没有数值要求，只要求非空。

目标是判断是否存在一种切法，使三段都满足要求。

## 第一反应

我一开始想到的是用左右计数维护：

```cpp
l1, l2, l3
r1, r2, r3
```

然后枚举一个位置 `i`，判断左边和右边是否合法。

这个思路的问题是：我只枚举了一个切点，实际上题目要求切成三段，需要两个切点。

我原来的思维相当于默认右段只有最后一个元素：

```text
[0 ... i] [i+1 ... n-2] [n-1]
```

但题目只是要求右段非空，右段可以有多个元素。

正确切法应该是：

```text
[1 ... x] [x+1 ... y] [y+1 ... n]
```

其中：

```text
1 <= x < y < n
```

所以这题的第一个坑不是算法，而是题意边界：右段不是固定最后一个元素。

## 核心思路

这题本质是一个三段切分问题。

暴力做法是枚举两个切点 `x` 和 `y`：

```text
[1 ... x] [x+1 ... y] [y+1 ... n]
```

要求：

```text
1 <= x < y < n
```

左段合法：

```text
cnt1 >= cnt2 + cnt3
```

中段合法：

```text
cnt1 + cnt2 >= cnt3
```

右段只要非空，不需要检查内容。

直接枚举两个切点是：

```text
O(n^2)
```

由于所有测试数据的 `n` 总和达到 `2e5`，所以需要优化。

关键转化是：把两个段的条件都转成前缀平衡值。

## 关键知识点

### 1. 左段条件转成前缀平衡值

左段要求：

```text
cnt1 >= cnt2 + cnt3
```

移项得到：

```text
cnt1 - cnt2 - cnt3 >= 0
```

定义：

```text
preL[i] = 前 i 个数中 cnt1 - cnt2 - cnt3
```

如果第一刀切在 `x`，左段就是前 `x` 个元素。

那么左段合法等价于：

```text
preL[x] >= 0
```

### 2. 中段条件转成前缀平衡值

中段要求：

```text
cnt1 + cnt2 >= cnt3
```

移项得到：

```text
cnt1 + cnt2 - cnt3 >= 0
```

定义：

```text
preM[i] = 前 i 个数中 cnt1 + cnt2 - cnt3
```

如果中段是：

```text
x+1 ... y
```

那么中段平衡值是：

```text
preM[y] - preM[x]
```

中段合法等价于：

```text
preM[y] - preM[x] >= 0
```

也就是：

```text
preM[x] <= preM[y]
```

### 3. 枚举第二刀，维护合法第一刀

我们需要找两个切点：

```text
1 <= x < y < n
```

满足：

```text
preL[x] >= 0
preM[x] <= preM[y]
```

枚举第二刀 `y`。

当枚举到当前 `y` 时，我们只需要知道在它前面是否存在一个第一刀 `x`，满足：

```text
preL[x] >= 0
preM[x] <= preM[y]
```

所以可以维护所有合法第一刀中的最小 `preM[x]`，记为 `best`。

如果：

```text
best <= preM[y]
```

说明存在某个合法第一刀 `x`，使得：

* 左段合法；
* 中段合法；
* 右段非空。

此时答案就是 `YES`。

第一，题意边界理解错了。

```text

```

```text
[左段] [中段] [任意非空右段]
```

所以原来的枚举漏掉了右段长度大于 `1` 的情况。

第二，我没有立刻想到把“两刀枚举”优化成“一刀枚举 + 历史信息维护”。

暴力枚举两个切点很自然，但优化时要意识到：

```text
固定 y 后，只需要知道前面是否存在合适的 x
```

然后把中段条件：

```text
preM[x] <= preM[y]
```

转成维护合法 `x` 中最小的 `preM[x]`。

这就是这题真正的转化点。

## 正确做法步骤

### 第一步：预处理两个前缀平衡值

`preL[i]` 用来判断左段是否合法：

```text
1 贡献 +1
2 贡献 -1
3 贡献 -1
```

`preM[i]` 用来判断中段是否合法：

```text
1 贡献 +1
2 贡献 +1
3 贡献 -1
```

### 第二步：枚举第二刀 y

第二刀 `y` 表示：

```text
[1 ... x] [x+1 ... y] [y+1 ... n]
```

所以必须满足：

```text
2 <= y <= n - 1
```

这样才能保证三段都非空。

### 第三步：把 x = y - 1 加入候选

当枚举到当前 `y` 时，新的候选第一刀是：

```cpp
int x = y - 1;
```

如果：

```text
preL[x] >= 0
```

说明前 `x` 个元素可以作为合法左段，于是把它的 `preM[x]` 纳入历史最小值：

```cpp
best = min(best, preM[x]);
```

### 第四步：判断是否存在合法第一刀

如果：

```text
best <= preM[y]
```

说明存在某个合法第一刀 `x`，满足：

```text
preL[x] >= 0
preM[x] <= preM[y]
```

也就是左段和中段都合法，右段也非空。

输出 `YES`。

如果所有 `y` 都不行，输出 `NO`。

## 代码实现要点

1. 这题是多测，要读 `t`。
2. 数组建议用 1 下标，方便表示前 `i` 个元素。
3. 枚举第二刀：

```cpp
for(int y = 2; y <= n - 1; ++y)
```

不能让 `y == n`，否则右段为空。

4. 当前新加入的第一刀是：

```cpp
int x = y - 1;
```

这样可以保证中段至少有一个元素。

5. `best` 表示所有合法第一刀中的最小 `preM[x]`。
6. 不要只保留第一个合法左段，因为后面的合法左段可能有更小的 `preM[x]`，更容易满足中段条件。

## 参考代码

```cpp
#include <bits/stdc++.h>
#define code using
#define by namespace
#define Mxiaocao std
code by Mxiaocao;

using ll = long long;

void solve(){
    int n; cin >> n;
    vector<int> a(n+1);
    for(int i = 1;i <= n;++i) cin >> a[i];

    vector<int> prel(n+1),prem(n+1);
    for(int i = 1;i <= n;++i){
        prel[i] = prel[i - 1];
        prem[i] = prem[i - 1];

        if(a[i] == 1){
            prel[i] += 1;
            prem[i] += 1;
        }else if(a[i] == 2){
            prel[i] -= 1;
            prem[i] += 1;
        }else{
            prel[i] -= 1;
            prem[i] -= 1;
        }
    }

    int best = 1e9;
    for(int y = 2;y <= n-1;++y){
        int x = y-1;
        if(prel[x] >= 0) best = min(best,prem[x]);
        if(prem[y] >= best){
            cout << "YES\n";
            return;
        }
    }
    cout << "NO\n";
    return;
}

signed main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int t = 1;
    cin >> t;
    while(t--){
        solve();
    }
    return 0;
}
```

## 本题总结

这题表面是三段切分，核心是：

```text
两刀枚举
→ 前缀平衡值转化
→ 枚举第二刀
→ 维护合法第一刀的历史最小 preM[x]
```

最重要的不是代码，而是这一步转化：

```text
中段合法：preM[y] - preM[x] >= 0
等价于：preM[x] <= preM[y]
```

所以只要枚举 `y`，维护所有合法 `x` 中最小的 `preM[x]` 即可。

以后遇到类似题目特征：

* 要切成多段；
* 每段有计数约束；
* 暴力需要枚举多个切点；
* 某一段条件可以写成前缀差；

就要想到：

```text
枚举一个切点，维护另一个切点的历史信息
```
