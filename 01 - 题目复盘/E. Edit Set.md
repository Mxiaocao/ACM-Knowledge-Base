## 题意简述

给定一个包含 $n$ 个不同整数的集合。

通过题目规定的编辑操作，可以让两个数最终变成相同的数。两个数相同以后，集合中的元素数量减少 $1$。

需要将集合大小从 $n$ 减少到 $k$，并最小化总操作代价。

## 核心模型

将数组从小到大排序：

$$
a_0<a_1<\cdots<a_{n-1}
$$

计算相邻差值：

$$
d_i=a_i-a_{i-1},\qquad 1\le i<n
$$

可以把排序后的元素看成一条链上的点：

```text
a0 --d1-- a1 --d2-- a2 -- ... -- a(n-1)
```

相邻差值就是链上的边权。

## 为什么只需要考虑相邻差值

假设一个最终合并组包含排序后的连续元素：

$$
a_l,a_{l+1},\dots,a_r
$$

连接这个连续区域所跨过的相邻差值为：

$$
a_{l+1}-a_l,\quad
a_{l+2}-a_{l+1},\quad
\dots,\quad
a_r-a_{r-1}
$$

这些差值之和为：

$$
(a_{l+1}-a_l)+\cdots+(a_r-a_{r-1})
$$

中间项相互抵消，最终得到：

$$
a_r-a_l
$$

因此，一个连续组内部的代价可以完全由相邻差值表示，不需要单独考虑非相邻元素之间的距离。

## 为什么选择 n-k 条边

初始时有 $n$ 个互相独立的元素。

每完成一次有效合并，集合大小减少 $1$。

要把集合大小从 $n$ 降低到 $k$，需要恰好：

$$
n-k
$$

次合并。

在链模型中，每选择一条边，就把两个相邻的独立部分连接起来，使连通块数量减少 $1$。

初始有 $n$ 个连通块，选择 $n-k$ 条边后，连通块数量为：

$$
n-(n-k)=k
$$

所以必须选择恰好 $n-k$ 条边。

## 为什么选择最小的 n-k 个差值

问题已经转化成：

> 从 $n-1$ 条相邻边中选择恰好 $n-k$ 条，使边权之和最小。

选择数量固定，每条边的代价独立相加。

因此直接选择权值最小的 $n-k$ 条边一定最优。

## 分块视角

也可以把最大的 $k-1$ 个相邻差值作为断点。

链中共有 $n-1$ 条边，删除 $k-1$ 条边后，剩余部分自然形成 $k$ 个连续块。

所以以下两种说法完全等价：

```text
选择最小的 n-k 条边
```

和：

```text
删除最大的 k-1 条边
```

分块只是帮助理解选边结果，不需要在代码中显式维护。

## 正确性证明

排序后的元素形成一条路径图。

将 $n$ 个独立元素减少到 $k$ 个元素，等价于让路径最终保留 $k$ 个连通块，因此必须选择 $n-k$ 条边。

每条被选择的边产生对应的相邻差值代价，总代价为所有被选边权之和。

由于选择数量固定为 $n-k$，选择权值最小的 $n-k$ 条边得到的总和最小。

因此算法正确。

## 为什么我会卡住

这题没有卡在代码，而是卡在证明。

我虽然观察到了：

```text
排序
→ 计算相邻差值
→ 排序差值
→ 选择较小差值
```

但没有立即把问题转化成链上的选边问题。

因此会困惑：

* 合并后元素位置变化怎么办；
* 为什么还能使用原始相邻差值；
* 连续块是否需要继续互相合并；
* 为什么不是直接使用整个区间的最大值减最小值。

真正的关键是：

```text
不要模拟合并后的具体元素位置，
只需要记录最终选择了哪些相邻边。
```

## 正确做法步骤

1. 对原数组排序；
2. 计算全部 $n-1$ 个相邻差值；
3. 将差值排序；
4. 从 $n$ 个元素减少到 $k$ 个元素，需要选择 $n-k$ 条边；
5. 累加最小的 $n-k$ 个相邻差值；
6. 输出答案。

## 代码实现要点

原代码使用了长度为 $n$ 的差值数组，导致其中包含一个额外的零，再人为从下标 $1$ 开始累加。

虽然能够通过，但不够直观。

更规范的写法是只保存真正存在的 $n-1$ 个相邻差值，并从下标 $0$ 开始累加前 $n-k$ 个。

答案建议使用 `long long`。

## 参考代码

```cpp
#include <bits/stdc++.h>
#define code using
#define by namespace
#define Mxiaocao std
code by Mxiaocao;

using ll = long long;

void solve(){
    int n,k;
    cin >> n >> k;

    vector<ll> a(n);
    for(auto &x : a) cin >> x;

    sort(a.begin(),a.end());

    vector<ll> d;
    for(int i = 1;i < n;++i){
        d.push_back(a[i]-a[i-1]);
    }

    sort(d.begin(),d.end());

    ll ans = 0;
    for(int i = 0;i < n-k;++i){
        ans += d[i];
    }

    cout << ans << '\n';
}

signed main(){
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    solve();
    return 0;
}
```

## 本题总结

本题的算法本质是：

```text
排序后形成一条链
→ 从链上选择 n-k 条边
→ 最小化所选边权之和
```

所以应该选择最小的 $n-k$ 个相邻差值。

等价地，也可以删除最大的 $k-1$ 条边，使链形成 $k$ 个连续块。
