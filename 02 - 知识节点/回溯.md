# 回溯

## 它是什么

回溯是一种通过尝试所有可能的选择，并在每次尝试后恢复状态的搜索方法。

回溯通常具有以下结构：

```text
做选择
→ 递归进入下一层
→ 撤销选择
```

核心特征：当前层的选择只影响当前递归分支，离开分支前需要恢复原状。

## 常见题目特征

- 需要枚举所有子集、排列、组合或路径。
- 每次决策影响后续可选项。
- 需要记录当前搜索路径。
- 可以使用剪枝提前排除无效分支。

## 选择、递归、撤销

排列枚举：

```cpp
void dfs(){
    if(path.size() == a.size()){
        output();
        return;
    }

    for(int i = 0;i < a.size();++i){
        if(used[i]){
            continue;
        }

        used[i] = true;
        path.push_back(a[i]);

        dfs();

        path.pop_back();
        used[i] = false;
    }
}
```

当前层决定排列中的一个位置。

`for` 循环枚举当前位置放哪个尚未使用的元素。

递归返回后撤销选择，是为了让同一层尝试其他分支。

## 如何判断状态是否恢复

判断一个状态要不要恢复，取决于它表达的含义。

### 当前搜索路径状态

例如：

- 排列中的 `used[i]`；
- 当前方案中的 `path`；
- 当前组合的 `sum`；
- 当前路径中是否使用某元素。

这些状态只属于当前递归分支。

离开分支后通常需要恢复。

### 全局处理状态

例如图 DFS 中：

```cpp
visited[u] = true;
```

它表示节点在整个遍历过程中已经被处理。

通常不能在返回时恢复，否则可能：

- 重复访问节点；
- 重复计算；
- 在环中无限递归。

因此不能只根据变量名判断是否恢复，必须先判断状态语义。

## 剪枝

剪枝是在已经能够证明当前分支不可能产生有效答案时提前返回。

例如所有数字非负时：

```cpp
if(sum > target){
    return;
}
```

只有当后续选择不可能让总和下降时，该剪枝才成立。

后缀和剪枝：

```cpp
if(sum + suffix[i] < target){
    return;
}
```

表示即使把剩余元素全部选择，也无法达到目标。

剪枝必须有正确性依据，不能只因为"看起来没希望"就停止。

## 常见应用

- 子集枚举；
- 全排列；
- 组合枚举；
- N 皇后；
- 数独求解；
- 带剪枝的子集和。
