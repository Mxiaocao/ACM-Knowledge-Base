# 归并排序

## 它是什么

归并排序使用分治策略，将数组分为左右两部分分别排序，再将两个有序区间合并为一个有序区间。

```text
排好左半部分
→ 排好右半部分
→ 合并两个有序区间
```

## 常见题目特征

- 需要对数组稳定排序。
- 需要统计逆序对。
- 需要合并两个有序序列。
- 需要在排序过程中利用左右两部分的有序性质。

## 双指针合并

合并时：

- `i` 指向左区间当前未使用元素；
- `j` 指向右区间当前未使用元素；
- `k` 指向临时数组下一处写入位置。

```cpp
while(i <= mid && j <= r){
    if(a[i] <= a[j]){
        temp[k++] = a[i++];
    }else{
        temp[k++] = a[j++];
    }
}
```

主循环使用 `&&`，因为只有左右两边都还有元素时才能继续比较。

退出后至少有一侧已经用完：

```cpp
while(i <= mid){
    temp[k++] = a[i++];
}

while(j <= r){
    temp[k++] = a[j++];
}
```

通常只有一个剩余循环真正复制元素。

最后必须复制回原数组，否则上一层看不到已经排好序的子区间。

## 复杂度分析

每次将区间长度减半，因此递归层数为：

$$
O(\log n)
$$

每一层所有区间长度之和仍然为 $n$，每层合并总工作量为：

$$
O(n)
$$

所以总时间复杂度为：

$$
O(n\log n)
$$

额外数组空间为：

$$
O(n)
$$

递归栈为：

$$
O(\log n)
$$

## 常见坑

1. `i、j、k` 的职责分不清；
2. 主循环误用 `||` 代替 `&&`；
3. 剩余元素复制时忘记下标范围；
4. 合并后没有复制回原数组，上一层看不到已排序结果；
5. 递归出口写成 `l >= r` 而不是 `l == r`，注意两者等价但语义上 `l == r` 更清晰代表区间长度为 1。
