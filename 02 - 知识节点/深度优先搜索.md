# 深度优先搜索

## 它是什么

深度优先搜索（DFS）是按深度优先顺序遍历或搜索树、图、网格等结构的方法。

DFS 不是单纯的"递归套一个 for 循环"，每种场景下的访问控制方式和状态管理有本质区别。

## 常见题目特征

- 需要在树、图或网格中按深度优先遍历。
- 需要统计连通块、子树大小、深度、可达性。
- 需要检测环或拓扑信息。
- 数据以节点和边的关系给出。

## 树上的 DFS

无根树通常使用父节点避免沿原边返回：

```cpp
void dfs(int u,int parent){
    for(int v : g[u]){
        if(v == parent){
            continue;
        }

        dfs(v,u);
    }
}
```

其中：

- `u`：当前节点；
- `parent`：当前节点的父节点；
- `dfs(v,u)`：进入子节点，并将当前节点作为它的父节点。

### 深度

```cpp
depth[v] = depth[u] + 1;
```

### 子树大小

```cpp
void dfs(int u,int parent){
    siz[u] = 1;

    for(int v : g[u]){
        if(v == parent){
            continue;
        }

        dfs(v,u);
        siz[u] += siz[v];
    }
}
```

必须先等待子节点返回，才能获得完整的 `siz[v]`。

## 一般图上的 DFS

一般图可能存在环，通常使用全局访问标记：

```cpp
void dfs(int u){
    visited[u] = true;

    for(int v : g[u]){
        if(visited[v]){
            continue;
        }

        dfs(v);
    }
}
```

`visited` 的作用包括：

- 防止在环中无限递归；
- 防止节点被重复处理；
- 保证每个节点最多进入一次 DFS。

**关键区分**：图 DFS 的 `visited` 不能被恢复。这与回溯中排列的 `used[i]` 在语义上完全不同。

## 连通块

统计连通块数量：

```cpp
int components = 0;

for(int i = 1;i <= n;++i){
    if(!visited[i]){
        dfs(i);
        components++;
    }
}
```

每次从一个未访问节点启动 DFS，就发现一个新的连通块。

统计连通块大小：

```cpp
int dfs(int u){
    visited[u] = true;

    int size = 1;

    for(int v : g[u]){
        if(visited[v]){
            continue;
        }

        size += dfs(v);
    }

    return size;
}
```

`size = 1` 先统计当前节点自己，再累加相邻未访问部分的大小。

## 网格 DFS

网格可以看成每个格子是一个节点的图。

四方向数组：

```cpp
int dx[4] = {-1,1,0,0};
int dy[4] = {0,0,-1,1};
```

访问相邻格子前通常检查：

```cpp
if(nx < 0 || nx >= n || ny < 0 || ny >= m){
    continue;
}

if(a[nx][ny] == 0){
    continue;
}

if(visited[nx][ny]){
    continue;
}
```

分别用于：

- 防止越界；
- 排除不能经过的格子；
- 防止重复访问和环。

常见应用：

- 岛屿数量；
- 连通区域数量；
- 区域大小；
- 最大连通区域；
- 迷宫可达性。

## 常见坑

1. 树 DFS 忘记 `parent` 参数导致沿原边返回；
2. 图 DFS 错误恢复 `visited` 导致重复访问或死循环；
3. 多测之间 `visited` 没有清空；
4. 网格 DFS 数组越界；
5. 连通块统计时忘记计算当前节点自己；
6. 混淆回溯的路径状态和图 DFS 的全局状态。
