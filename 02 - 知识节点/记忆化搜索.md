# 记忆化搜索

## 它是什么

记忆化搜索在普通递归的基础上，将已经计算过的状态答案存储下来，避免重复计算相同状态。

## 常见题目特征

- 普通递归存在大量重复状态。
- 状态空间有限且可以枚举。
- 每个状态的答案只依赖更小的状态。
- 状态转移有明确的依赖方向。

## 从重复计算到记忆化

普通斐波那契递归：

```cpp
long long fib(int n){
    if(n <= 2){
        return 1;
    }

    return fib(n - 1) + fib(n - 2);
}
```

相同状态会被多次计算，时间复杂度接近：

$$
O(2^n)
$$

关键问题不是递归本身慢，而是重复计算了相同状态。

## 记忆化实现

```cpp
long long fib(int n){
    if(n <= 2){
        return 1;
    }

    if(memo[n] != -1){
        return memo[n];
    }

    memo[n] = fib(n - 1) + fib(n - 2);

    return memo[n];
}
```

正确顺序是：

```text
检查缓存
→ 命中则直接返回
→ 未命中才递归
→ 保存答案
→ 返回答案
```

真正保存答案的是：

```cpp
memo[n] = ...
```

而不是检查语句。

如果状态共有 $n$ 个，每个状态只真正计算一次，时间复杂度为：

$$
O(n)
$$

## 记忆化复杂度

记忆化搜索通常使用：

$$
\text{状态数量}\times\text{每个状态的转移数量}
$$

估算时间复杂度。

例如网格中状态为坐标：

```cpp
dfs(x,y)
```

共有：

$$
n\times m
$$

个状态。

如果每个状态只转移到右边和下面，转移数量为常数，总复杂度为：

$$
O(nm)
$$

## 从记忆化到递推 DP

记忆化搜索：

- 从目标状态出发；
- 需要哪个子状态才计算哪个；
- 属于自顶向下。

递推 DP：

- 从已知基础状态出发；
- 按依赖顺序计算后续状态；
- 属于自底向上。

斐波那契递推：

```cpp
dp[1] = 1;
dp[2] = 1;

for(int i = 3;i <= n;++i){
    dp[i] = dp[i - 1] + dp[i - 2];
}
```

遍历顺序必须保证依赖状态已经计算。

## 二维记忆化

定义：

```text
dfs(x,y)：从 (x,y) 出发到终点的最优答案。
```

如果只能向右或向下：

```cpp
dfs(x,y) = a[x][y] + max(
    dfs(x + 1,y),
    dfs(x,y + 1)
);
```

非法状态在最大值问题中应返回足够小的值，避免被选中。

记忆化状态使用：

```cpp
memo[x][y]
```

因为子问题由行和列共同确定。

递推形式必须按照依赖顺序计算。

如果当前位置依赖右边和下面，就应从右下向左上遍历。

## 空间压缩

当当前状态只依赖有限个历史状态时，可以压缩存储。

```cpp
long long prev2 = 1;
long long prev1 = 1;

for(int i = 3;i <= n;++i){
    long long current = prev1 + prev2;

    prev2 = prev1;
    prev1 = current;
}
```

更新顺序的重点是防止旧状态被提前覆盖。

压缩前必须先明确：

- 当前状态依赖哪些旧状态；
- 哪些旧状态之后不再需要；
- 更新变量是否会覆盖仍需读取的值。

## 常见坑

1. 只检查缓存但没有保存答案；
2. `memo` 初始值与合法答案冲突（例如用 `-1` 表示未计算，但合法答案也可能是 `-1`）；
3. 混淆记忆化搜索（自顶向下）和递推 DP（自底向上）的遍历方向；
4. 空间压缩时覆盖了仍需读取的旧状态；
5. 状态定义不明确导致 `memo` 维度错误。
