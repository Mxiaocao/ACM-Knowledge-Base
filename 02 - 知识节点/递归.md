# 递归

## 它是什么

递归是函数直接或间接调用自身，用规模更小的同类问题解决当前问题的方法。

递归不是单纯的"函数调用自己"，一个可以正确终止的递归通常需要同时明确：

1. 当前递归函数表示什么；
2. 参数表示什么状态；
3. 返回值表示什么答案；
4. 最小问题是什么；
5. 当前问题如何转化成更小的问题；
6. 子问题返回后，当前层还要做什么。

## 常见题目特征

- 问题可以分解为规模更小的同类子问题。
- 状态空间具有自然的嵌套或层次结构。
- 需要枚举所有可能的选择或路径。
- 树、图等递归结构上的遍历和计算。
- 数据范围支持指数级枚举或对数级分治。

## 设计递归前先回答什么

写递归函数之前，应先用一句话定义函数含义。

例如：

```text
getSum(l,r)：返回数组下标 l 到 r 的元素和。
```

```text
dfs(u)：访问从节点 u 出发能够到达的全部未访问节点。
```

```text
dfs(u,parent)：处理以 u 为根、parent 为父节点的子树。
```

```text
dfs(x,y)：返回从 (x,y) 出发到终点的最优答案。
```

如果函数含义不清楚，通常会继续出现：

- 出口不知道返回什么；
- 参数不知道如何变化；
- 返回值与函数定义不一致；
- 当前层与子问题职责混乱；
- 代码能够模仿，却无法独立修改。

## 递归的基本组成

### 函数含义

明确当前调用负责解决什么问题。

### 递归出口

出口负责处理不能或不需要继续缩小的最小状态。

例如阶乘：

```cpp
long long factorial(int n){
    if(n == 1){
        return 1;
    }

    return n * factorial(n - 1);
}
```

### 规模缩小

每次递归调用必须让问题接近出口。

例如：

```cpp
factorial(n - 1)
```

如果参数没有变小、状态没有推进，可能形成无限递归。

### 使用子问题答案

子问题返回后，当前层使用返回值构造自己的答案。

例如区间最大值：

```cpp
int getMax(int l,int r){
    if(l == r){
        return a[l];
    }

    int mid = (l + r) / 2;

    int leftMax = getMax(l,mid);
    int rightMax = getMax(mid + 1,r);

    return max(leftMax,rightMax);
}
```

## 调用和返回顺序

执行递归调用时，当前函数不会消失。

当前层会：

1. 保存局部变量和当前执行位置；
2. 暂停；
3. 进入下一层调用；
4. 等待下一层返回；
5. 从递归调用的下一行继续执行。

例如：

```cpp
void dfs(int n){
    cout << n << ' ';

    if(n == 1){
        return;
    }

    dfs(n - 1);

    cout << n << ' ';
}
```

递归调用前的代码按外层到内层执行。

递归调用后的代码按内层到外层执行。

这不是需要死记的规则，而是因为外层函数一直暂停在递归调用处，必须等内层返回后才能继续。

## 递归调用栈

每一次函数调用都拥有独立的：

- 参数；
- 局部变量；
- 循环变量；
- 返回地址；
- 当前执行位置。

例如排列递归中，每层都有自己的 `i`。

下一层重新执行：

```cpp
for(int i = 0;i < n;++i)
```

不会改变上一层暂停时保存的 `i`。

当下一层返回后，上一层继续自己的循环。

## 单链递归

如果每个状态只调用一个更小状态，就形成单链递归。

例如数组求和：

```cpp
long long getSum(int i){
    if(i == n){
        return 0;
    }

    return a[i] + getSum(i + 1);
}
```

常见问题：

- 阶乘；
- 数组求和；
- 数组最大值；
- 查找元素；
- 判断有序；
- 双指针递归处理区间。

如果递归深度为 $n$，每层工作为 $O(1)$，通常时间复杂度为：

$$
O(n)
$$

递归栈空间通常为：

$$
O(n)
$$

## 分支递归

一个状态产生多个递归调用时，会形成递归树。

例如二进制选择：

```cpp
dfs(i + 1);

choose(i);
dfs(i + 1);
undo(i);
```

两次递归调用表示两个不同选择：

- 不选择当前元素；
- 选择当前元素。

如果每层有两个分支、深度为 $n$，状态数量通常可能达到：

$$
O(2^n)
$$

## 复杂度判断

### 单链递归

深度 $n$，每层 $O(1)$：

$$
O(n)
$$

### 单链递归中带递减循环

如果总工作量为：

$$
n+(n-1)+\cdots+1
$$

则为：

$$
O(n^2)
$$

### 二叉分支递归

没有记忆化时可能达到：

$$
O(2^n)
$$

### 子集

状态数量：

$$
O(2^n)
$$

如果每个叶子输出整个集合：

$$
O(n2^n)
$$

### 全排列

排列数量：

$$
O(n!)
$$

如果每个排列输出 $n$ 个元素：

$$
O(n\cdot n!)
$$

### 归并排序

$$
O(n\log n)
$$

### 记忆化搜索

$$
\text{状态数量}\times\text{每状态转移数量}
$$

## 常见坑

1. 没有先定义函数含义；
2. 出口与返回值含义不一致；
3. 子问题规模没有变小；
4. 递归返回后忘记当前层仍会继续执行；
5. 混淆不同递归层的局部变量；
6. 回溯状态忘记恢复；
7. 图 DFS 的 `visited` 被错误恢复；
8. 多测之间没有清空全局状态；
9. 数组或网格越界；
10. 非法状态返回值可能被当成合法答案；
11. 递归深度过大导致栈溢出；
12. `int` 发生溢出；
13. 记忆化只检查但没有保存；
14. `memo` 初始值与合法答案冲突；
15. DP 遍历顺序不满足依赖；
16. 空间压缩时提前覆盖旧状态；
17. 归并排序后没有复制回原数组；
18. 合并时指针范围写错；
19. 只会背 DFS 模板，却说不清函数定义。
