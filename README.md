# ACM Knowledge Base

这是一个用于 ACM 训练的 Obsidian 知识库。仓库目标是把每日训练、题目复盘、知识节点、比赛复盘和二刷计划连接成可长期维护的知识网络。

## 仓库目标

- 记录每天做过什么题、卡在哪里、暴露了什么问题。
- 将题目连接到对应知识节点，形成可复用的知识链路。
- 维护 ACM 常见算法、数据结构和思维方法的知识地图。
- 通过二刷清单追踪薄弱题和薄弱知识点。
- 使用 Git 做长期备份，避免训练资料丢失。

## 每天训练流程

1. 在 `90 - 原始材料` 中记录当天原始做题过程。
2. 对需要沉淀的题，在 `01 - 题目复盘` 中使用 `04 - 模板/单题复盘模板.md` 创建题目笔记。
3. 在题目笔记中补充主链路和副链路，统一使用 `[[知识点名]]`。
4. 如果出现新的知识点，优先检查 `02 - 知识节点` 中是否已有节点。
5. 在 `00 - 每日复盘` 中使用 `04 - 模板/每日复盘模板.md` 总结当天问题。
6. 需要二刷的题加入 `06 - 二刷清单/二刷总表.md`。

## 文件夹说明

- `00 - 每日复盘`：每日训练总结、问题诊断和明日计划。
- `01 - 题目复盘`：单题复盘笔记，后续按“来源 - 题名”命名。
- `02 - 知识节点`：算法、数据结构、数学、思维方法等知识点节点。
- `03 - 比赛复盘`：比赛整体表现、失误、补题和后续注意事项。
- `04 - 模板`：题目、每日、知识节点和比赛复盘模板。
- `05 - 知识地图`：高层知识网络和学习路线。
- `06 - 二刷清单`：需要再次训练的题目列表。
- `90 - 原始材料`：未经整理的做题记录、错误记录和官方题解摘录。
- `99 - 附件`：图片、截图、PDF 等附件。

## 题目复盘流程

1. 先记录题目基本信息、题意简述和第一反应。
2. 再写核心思路，但不要让 AI 擅自替换教练给出的核心判断。
3. 用主链路描述最关键的知识路径。
4. 用副链路记录相关但非主线的知识点。
5. 写清楚自己为什么卡住，以及正确做法步骤。
6. 最后补充实现要点、复杂度、总结和二刷提醒。

## 知识节点维护规则

- 一个知识点只维护一个主节点，避免同义重复。
- 新节点要能被 `05 - 知识地图/ACM 总知识地图.md` 或其他节点链接到。
- 相关节点分为上位节点、下位节点和交叉节点。
- 相关题目只记录真实做过或明确复盘过的题。
- 知识节点先短后长，随着题目复盘逐步补充。

## Git 备份规则

- 每天训练结束后提交一次 Git。
- 提交前检查是否有误删文件。
- 提交信息建议写清楚当天新增内容，例如：`新增 2026-07-06 每日复盘和二刷清单`。
- 不要把临时文件、无关下载文件和大体积无关材料加入仓库。
